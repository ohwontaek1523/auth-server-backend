## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Environment Setup

이 프로젝트는 환경별 `.env` 파일을 사용합니다:
- `.env.development` - 개발 환경
- `.env.production` - 운영 환경

`dotenv-cli`를 사용하여 명령어 실행 시 자동으로 해당 환경의 파일을 로드합니다.

## Compile and run the project

```bash
# development (자동으로 .env.development 적용)
$ npm run start:dev

# production (자동으로 .env.production 적용)
$ npm run start:prod

# debug mode
$ npm run start:debug
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Database & Prisma

### Prisma 스키마 관리

**중요**: DB에서 스키마를 가져올 때 `prisma-case-format`이 자동으로 snake_case를 camelCase로 변환하고 `@map`을 추가합니다.

```bash
# 개발 DB에서 스키마 가져오기 (snake_case → camelCase 자동 변환)
$ npm run db:pull:dev

# 운영 DB에서 스키마 가져오기
$ npm run db:pull:prod

# Prisma Studio 실행
$ npm run prisma:studio:dev     # 개발 DB
$ npm run prisma:studio:prod    # 운영 DB

# Prisma Client 재생성
$ npm run prisma:generate
```

### 주의사항

- 각 명령어는 자동으로 해당 환경의 `.env` 파일을 로드합니다
- `.env` 파일은 Git에 커밋하지 마세요 (`.gitignore`에 포함)
- Prisma 명령어도 환경별로 분리되어 있으니 주의하세요
